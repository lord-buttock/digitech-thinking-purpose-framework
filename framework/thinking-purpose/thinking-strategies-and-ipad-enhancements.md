# Thinking Strategies, Student Roles And iPad Enhancements

> Meaningful iPad use starts with the thinking, not the app.

Teachers first choose the thinking-purpose category, then the thinking strategy, then the student role, and only then decide whether the iPad adds genuine learning value.

The core planning chain is:

```text
Thinking-purpose category -> thinking strategy -> student role -> iPad enhancement -> artefact -> evidence of learning
```

This page defines the reusable thinking-purpose categories and student roles for the whole-school Meaningful iPad Use framework. They are not a sequence, and every task does not need every role. Teachers select the category and roles that best match the learning purpose.

The framework helps shift iPad use from:

> students using an app

to:

> students doing meaningful thinking with a tool.

## Thinking-Purpose Categories

These categories are a **purpose selector**, not a planning sequence. Teachers choose the category that best matches what students need to do with their thinking.

| Thinking-Purpose Category                      | When The Teacher Wants Students To…                                                                             | Main Student Roles                              |
| ---------------------------------------------- | --------------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| **Notice, Wonder And Explore**                 | Observe closely, ask questions, gather initial evidence, notice patterns and begin inquiry.                     | Observers, researchers                          |
| **Compare, Classify And Analyse**              | Sort, group, compare, contrast, identify patterns, examine similarities/differences and analyse features.       | Analysers, observers, researchers               |
| **Investigate, Connect And Explain**           | Connect evidence to claims, explain relationships, show cause/effect, describe processes and justify reasoning. | Explainers, researchers, collaborators          |
| **Explore Objects, Systems And Design**        | Examine parts, purposes, functions, structures, systems, design choices and how things work.                    | System thinkers, designers, explainers          |
| **Explore Perspectives And Impacts**           | Consider viewpoints, stakeholders, values, consequences, fairness, change and impact.                           | Perspective-takers, researchers, storytellers   |
| **Imagine, Design And Improve**                | Generate possibilities, prototype, test, improve, redesign and solve problems creatively.                       | Designers, creators, collaborators              |
| **Code, Model And Simulate**                   | Build interactive products, test rules, debug, model systems, simulate cause/effect or processes.               | Coders/builders, designers, system thinkers     |
| **Organise, Summarise And Show Understanding** | Synthesise knowledge, communicate understanding, create summaries, teach others and present learning clearly.   | Communicators, explainers, storytellers         |
| **Reflect, Evaluate And Improve**              | Evaluate quality, respond to feedback, show growth, revise work and explain how thinking changed.               | Reflectors, evaluators, self-regulated learners |

## How The Categories Work

The categories help teachers start with the learning purpose instead of the app.

For example:

| If Students Need To... | The Teacher Might Choose... |
|---|---|
| slow down, notice details and ask better questions | **Notice, Wonder And Explore** |
| look for patterns, similarities and differences | **Compare, Classify And Analyse** |
| connect evidence to a claim or explain how something works | **Investigate, Connect And Explain** |
| consider stakeholders, viewpoints and impacts | **Explore Perspectives And Impacts** |
| revise work after feedback and explain growth | **Reflect, Evaluate And Improve** |

The categories can overlap. A strong task may involve more than one kind of thinking, but teachers should name the **main thinking purpose** so the iPad use stays focused.

## Thinking Strategy Suitability Matrix

They are not apps. They are not extra decoration. A thinking strategy is useful when it helps students understand what kind of thinking they are doing and produces evidence the teacher can respond to.

### Strategy Selection Rule

Choose the thinking strategy that best matches the learning purpose and the curriculum thinking demand.

For example:

- If students need to **notice carefully before interpreting**, use **See, Think, Wonder** or **Looking Ten Times Two**.
- If students need to **compare examples and identify category rules**, use **Same and Different**, **Venn Diagram** or **Generate, Sort, Connect, Elaborate**.
- If students need to **justify an idea using evidence**, use **Claim, Support, Question** or **What Makes You Say That?**
- If students need to **understand a system**, use **Parts, Purposes, Complexities** or a systems map.
- If students need to **revise after feedback**, use **I Used To Think, Now I Think**, **Compass Points** or criteria-based reflection.

If the thinking strategy does not produce stronger learning evidence, simplify the task.

## Student Roles

Student roles describe what students are practising as learners. They help teachers name what students are actually doing with the iPad beyond completing a digital task.

| Student Role | Descriptor |
|---|---|
| **Students as observers** | Students look closely, notice details, capture evidence and describe what they see, hear, feel, measure or experience. |
| **Students as questioners** | Students generate, improve and select questions that are connected to evidence, curiosity, uncertainty, a text, a problem or a learning goal. |
| **Students as researchers** | Students gather, select and check evidence, examples, data, observations, sources, vocabulary, mentor texts or information to build understanding. |
| **Students as analysers** | Students compare, sort, classify, identify patterns, examine features, notice relationships and look for what is similar, different, important or surprising. |
| **Students as explainers** | Students make their thinking visible by explaining how, why or what they know, using evidence, examples, strategies, vocabulary or reasoning. |
| **Students as problem-solvers** | Students work through challenges, test strategies, make decisions, try alternatives, justify solutions and learn from errors or feedback. |
| **Students as system thinkers** | Students examine how parts, rules, structures, processes or relationships work together in a whole, such as a text, number system, ecosystem, machine, community or organisation. |
| **Students as perspective-takers** | Students consider viewpoints, experiences, values, stakeholders, fairness, impacts and consequences before forming or revising an idea. |
| **Students as designers** | Students plan, shape, test and improve texts, products, models, explanations, investigations, representations, environments or solutions for a purpose. |
| **Students as creators** | Students make purposeful artefacts that combine ideas, evidence, media, voice, design, vocabulary and audience to show or extend learning. |
| **Students as storytellers** | Students use sequence, narrative, voice, image, sound, movement, data or process to communicate meaning, change, experience or understanding. |
| **Students as collaborators** | Students build shared understanding by taking roles, contributing ideas, listening, comparing evidence, challenging respectfully and revising together. |
| **Students as builders/coders** | Students construct, program, model, simulate, debug or test interactive systems, physical builds, digital products, algorithms or prototypes. |
| **Students as communicators** | Students organise and present ideas clearly for a specific audience, purpose and format, using appropriate language, structure and media. |
| **Students as reflectors** | Students evaluate quality, respond to feedback, track growth, identify next steps and explain how their thinking, strategy or product changed. |

## Role Selection Rule

A strong iPad task usually names one or two main student roles. If too many roles are listed, the task may be too broad or unclear.

For example, in **Notice, Wonder And Explore**, the most likely core roles are:

- Students as observers
- Students as questioners
- Students as researchers
- Students as collaborators

Sometimes this category also includes:

- Students as storytellers, when they use voice, image or video to share early thinking.

## Digital Functions and Their Educational Value

These are the specific iPad affordances that can enhance learning tasks. Each function earns its place when it supports a thinking purpose; use the Strategy Suitability Matrix above to match function to category.

| Digital Function        | Educational Enhancement                                                                              |
| ----------------------- | ---------------------------------------------------------------------------------------------------- |
| Camera                  | Captures real-world evidence, places, objects, processes and student work.                           |
| Zoom / crop / close-up  | Reveals details students may miss with normal observation.                                           |
| Slow motion             | Makes fast movement visible and replayable.                                                          |
| Time-lapse              | Makes slow change visible over time.                                                                 |
| Audio recording         | Captures oral reasoning, questions, reflection, fluency and student voice.                           |
| Video recording         | Captures process, movement, explanation, performance and change.                                     |
| Annotation / markup     | Lets students label, highlight, connect and explain directly on evidence.                            |
| Multimodal composition  | Combines image, text, voice, video, drawing, links and layout.                                       |
| Revision / duplication  | Lets students revise, compare versions and show how thinking changed.                                |
| Sharing / gallery       | Allows groups to compare work, build class knowledge and receive feedback.                           |
| Links / sources         | Connects ideas to websites, articles, videos, maps or references.                                    |
| Interactivity           | Lets users tap, choose, navigate, explore or respond.                                                |
| Augmented reality       | Places models or designs in real space to explore scale, function or context.                        |
| 3D modelling            | Allows students to design, test and refine spatial objects.                                          |
| Simulation / coding     | Lets students model systems, test rules, debug and see consequences.                                 |
| Data handling           | Supports collecting, sorting, visualising and interpreting data.                                     |
| Accessibility           | Supports students through voice, captions, translation, enlarged text, audio and visual scaffolds.   |
| Publishing              | Creates artefacts for real audiences beyond the classroom.                                           |

## Database-Ready Field Set

The framework should be written so it can later become a searchable website, spreadsheet, Airtable-style database or planning assistant.

This does not mean teachers need to see every field all at once. It means each category guide and activity card should use consistent information underneath the surface.

### Category Guide Fields

Use these fields for each thinking-purpose category guide.

| Field | Purpose |
|---|---|
| **Thinking-purpose category** | The main framework category and primary lookup filter. |
| **Category summary** | A short teacher-friendly explanation of what this category is for. |
| **Memorable anchor** | A simple phrase teachers can remember and repeat. |
| **Thinking job** | The kind of thinking students are doing in plain language. |
| **Best used when** | The classroom moments, unit phases or learning needs that suit the category. |
| **Core student roles** | The roles that are usually central to this category. |
| **Supporting student roles** | Roles that may apply depending on the subject or task. |
| **Core thinking strategies** | The main routines, organisers or scaffolds that support the category. |
| **Supporting thinking strategies** | Useful secondary strategies that may fit particular subjects or tasks. |
| **Strong iPad artefacts** | The digital artefact types that are usually meaningful for this category. |
| **iPad enhancement rule** | The simple rule for when the iPad is worth using. |
| **Substitution danger zone** | Common weak uses that should be avoided or redesigned. |
| **Required evidence of learning** | The assessable traces of thinking the task should produce. |
| **Victorian Curriculum fit** | Genuine curriculum areas, strands or capabilities served by the category. |
| **VTLM 2.0 fit** | How the category supports explicit teaching and supported application. |
| **Framework checks** | SAMR, PICRAT, EdTech Quintet and TPACK guidance. |
| **Busy teacher version** | A concise planning summary for quick use. |
| **Leadership rationale** | Principal-friendly wording for why this category supports meaningful digital learning. |

### Activity Card Fields

Use these fields for individual examples or activities.

| Field | Purpose |
|---|---|
| **Activity name** | A short, memorable name teachers can recognise. |
| **Thinking-purpose category** | Main category filter. |
| **Year band** | F-2, 3-4 or 5-6. |
| **Curriculum area** | Reading, Writing, Maths, Science, Humanities, The Arts, Wellbeing, Digital Technologies or Integrated Studies. |
| **Learning intention** | What students are meant to learn, understand or practise. |
| **Student roles** | One or two main roles, with optional supporting roles. |
| **Thinking strategy** | The routine, organiser or scaffold used to structure the thinking. |
| **iPad artefact** | What students create, capture or produce. |
| **Best tool or format** | The practical iPad setup, such as Camera, Keynote, Freeform, Clips, Voice Memos, Canva, Pages, ScratchJr or Safari. |
| **Device model** | One iPad per group, small set, class set, teacher iPad, or rotating station. |
| **Setup complexity** | Low, medium or high. This helps busy teachers choose realistic options. |
| **Teacher setup** | What the teacher prepares before students begin. |
| **What students do** | The concrete student actions, step by step. |
| **Traditional version** | What the non-digital version would be. |
| **Digital function used** | The specific iPad affordance, such as camera, zoom, replay, audio, annotation, simulation or publishing. |
| **Enhancement type** | Evidence capture, close observation, student voice, replay, revision, collaboration, simulation, audience, accessibility, feedback or another genuine enhancement. |
| **iPad enhancement decision** | Keep, optional, redesign or use non-digital method. |
| **Why not paper? answer** | The explicit explanation of why the iPad adds learning value. |
| **Evidence of learning** | What the teacher can assess or respond to. |
| **Curriculum links** | Specific Victorian Curriculum links where they genuinely fit. |
| **VTLM 2.0 fit** | What is explicitly taught first and how the iPad supports application. |
| **SAMR fit** | Substitution, Augmentation, Modification or Redefinition. |
| **PICRAT fit** | Passive/Interactive/Creative and Replace/Amplify/Transform. |

### Database Design Principle

The database should help teachers answer:

> What do I want students to do with their thinking, and when is the iPad genuinely worth using?

It should not become an app menu.

The future website should make it easy for teachers to search by curriculum area, year band or practical classroom need, but the framework logic should still privilege:

```text
learning purpose -> thinking strategy -> student role -> iPad enhancement -> evidence
```
